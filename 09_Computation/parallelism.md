计算图（Computational Graph）
============================

计算图是用图结构表示数学运算过程的抽象模型，回答了深度学习框架的两个核心问题：如何计算（前向）与如何求导（反向）。

1. 基本构成
  - 节点：操作（如加法、乘法）或变量/常数（输入、中间结果、输出）
  - 边：数据流动方向，表示依赖关系

示例：y = (a + b) * c 的计算图
  a 和 b 指向加法节点 +，得到中间结果 t = a + b；t 和 c 指向乘法节点 *，得到最终输出 y = t * c。

2. 自动求导（反向传播）
计算图通过沿反向边应用链式法则实现自动求导。
对于 y = t * c，t = a + b：
  ∂y/∂a = (∂y/∂t) * (∂t/∂a) = c * 1 = c
反向传播时，每个节点只需计算局部导数，乘以上游梯度后向下游传递。

3. 动态图 vs 静态图
+----------------+----------------------------+----------------------------------+
| 特性           | 动态计算图（PyTorch Eager）  | 静态计算图（TorchScript/TF1.x） |
+----------------+----------------------------+----------------------------------+
| 构建时机       | 运行时，逐行构建            | 运行前，完整定义                 |
| 灵活性         | 极高，支持原生控制流（if/while）| 较低，需受限语法                |
| 调试体验       | 像普通 Python 一样用 print   | 需要专用工具，较困难             |
| 执行效率       | 较低（Python 解释器逐层开销）| 高（图优化后以 C++ 执行）        |
| 典型用途       | 研究、原型开发、动态网络      | 生产部署、多 GPU 加速            |
+----------------+----------------------------+----------------------------------+

混合式编程（如 torch.jit.script）先用动态图编写和调试，再一键转换成静态图。

4. PyTorch 中的计算图示例
  a = torch.tensor(2.0, requires_grad=True)
  b = torch.tensor(3.0, requires_grad=True)
  c = torch.tensor(4.0)
  t = a + b   # 加法节点
  y = t * c   # 乘法节点
  y.backward()
  
执行过程：
  - 前向动态构建 DAG。
  - 调用 backward() 后反向传播：y 的梯度为1，到乘法节点：传递给 t 的梯度为 c=4，传递给 c 的梯度为 t=5（c无梯度）。到加法节点：传递给 a,b 的梯度均为 4*1=4。
  - a.grad = 4.0，b.grad = 4.0。
  - 默认反向后释放计算图。

5. 与多 GPU 并行的关系
  - 计算图显式记录数据依赖。
  - 互不依赖的操作可并行执行。
  - 相互依赖的操作必须串行。
  - 静态图允许提前优化，消除 Python 解释器瓶颈，提升多 GPU 利用率。

一句话总结：计算图是数学公式的“路线图”：节点表示运算或数据，边表示依赖关系。它让计算机既能正确顺序执行前向计算，又能通过反向传播自动求导。动态图灵活易调，静态图高效可部署。
