# Seq2Seq 模型总结

## 1. 编码器（Encoder）

- **作用**：将长度可变的输入序列转换为固定形状的上下文向量 $\mathbf{c}$。
- **实现**：通常使用 RNN（单向或双向）。每个时间步 $t$ 更新隐状态：
  \[
  \mathbf{h}_t = f(\mathbf{x}_t, \mathbf{h}_{t-1})
  \]
- **上下文向量**：通过聚合函数 $q$ 得到 $\mathbf{c} = q(\mathbf{h}_1, \dots, \mathbf{h}_T)$。最简单的取法是 $\mathbf{c} = \mathbf{h}_T$。
- **双向 RNN**：能同时利用过去和未来的信息，得到更丰富的表示。

## 2. 解码器（Decoder）

- **作用**：基于上下文向量 $\mathbf{c}$ 自回归地生成目标序列 $y_1, \dots, y_{T'}$。
- **隐状态更新**：
  \[
  \mathbf{s}_t = f(\mathbf{y}_{t-1}, \mathbf{s}_{t-1}, \mathbf{c})
  \]
- **输出概率**：
  \[
  P(y_t \mid y_{<t}, \mathbf{c}) = \text{softmax}(\mathbf{W}_o \mathbf{s}_t + \mathbf{b}_o)
  \]

## 3. 损失函数：带掩码的交叉熵

- **基本形式**（负对数似然）：
  \[
  \mathcal{L}_{\text{NLL}} = -\sum_{t=1}^{T'} \log \hat{y}_t[y_t]
  \]
- **处理变长序列（填充 + 掩码）**：
  - 将批次内序列填充到相同长度 $L_{\max}$，引入掩码 $m_t \in \{0,1\}$（1 表示有效位置）。
  - 最终损失：
    \[
    \mathcal{L} = -\frac{1}{\sum_t m_t} \sum_{t=1}^{L_{\max}} m_t \log \hat{y}_t[y_t]
    \]

## 4. 教师强制（Teacher Forcing）

- **定义**：训练时，解码器每一步的输入使用**真实的上一个目标词元** $y_{t-1}$，而非模型自己的预测。
- **优点**：加速收敛，训练稳定。
- **潜在问题**：暴露偏差（训练与推理输入分布不一致），可通过计划采样、强化学习等缓解。

## 5. 评估指标：BLEU

- **核心思想**：通过统计预测序列与标签序列中 $n$-gram 的匹配程度来评估生成质量。
- **定义**：
  \[
  \text{BLEU} = \exp\left(\min\left(0, 1 - \frac{\text{len}_{\text{label}}}{\text{len}_{\text{pred}}}\right)\right) \prod_{n=1}^k p_n^{\,1/2^n}
  \]
  - $p_n$：$n$-gram 的精确度（匹配的 $n$-gram 数 / 预测中 $n$-gram 总数）。
  - 长度惩罚项：防止预测序列过短。
- **值域**：0 ~ 1，完全匹配时为 1。

## 6. 关键点总结

| 组件 | 功能 | 备注 |
|------|------|------|
| 编码器 | 将输入序列编码为上下文向量 | 无独立损失，通过解码器反向传播更新 |
| 解码器 | 自回归生成目标序列 | 训练时使用教师强制 |
| 损失函数 | 带掩码的交叉熵 | 忽略填充位置，平均有效位置的负对数似然 |
| BLEU | 评估生成序列与参考序列的相似度 | 基于 $n$-gram 匹配，包含长度惩罚 |

> Seq2Seq 模型通过编码器-解码器结构实现变长序列到变长序列的映射，是机器翻译、文本摘要等任务的基础。
