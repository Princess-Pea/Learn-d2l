专业术语 (CN)	英文全称 (EN)	简要描述 (Description)
具身智能	Embodied AI	强调智能必须具备物理实体（身体），通过与环境的交互来学习，而非仅仅处理离线数据。
感知 - 动作循环	Perception-Action Loop	智能体感知环境、做出决策、执行动作、观察新状态的闭环过程。
RGB-D 相机	RGB-Depth Camera	同时采集彩色图像和深度信息的相机，通常利用红外技术测量距离。
飞行时间法	ToF (Time of Flight)	通过计算红外脉冲发射与返回的时间差来测量距离的技术。
结构光	Structured Light	向物体投射特定图案（散斑），根据图案畸变计算物体 3D 形状的技术。
主动双目视觉	Active Stereo Vision	模拟人眼视差，并主动投射红外光以解决无纹理物体（如白墙）匹配难题的技术。
点云	Point Cloud	空间中海量 3D 坐标点的集合，能精确还原物体的三维几何形状和绝对尺寸。
尺度歧义	Scale Ambiguity	2D 图像中 “近大远小” 导致无法判断物体真实尺寸的现象，3D 感知可消除此问题。
视差	Disparity	同一物体在左右两个相机成像平面上的位置差异，是计算深度的核心。
GelSight 触觉	GelSight Tactile Sensing	将触觉转化为图像的一种技术：通过摄像头拍摄受压硅胶表面的微小形变。

物理引擎	Physics Engine	模拟器中负责计算重力、碰撞、摩擦力等物理规律的核心软件模块（如 PhysX）。
刚体 / 多体动力学	Rigid/Multi-body Dynamics	模拟不发生形变的坚硬物体及多个连接零件（如机械臂）受力与运动的学科。
大规模并行仿真	Large-scale Parallel Simulation	利用 GPU 同时运行成千上万个模拟环境，极大地加速数据采集和训练速度。
Sim-to-Real Gap	Sim-to-Real Gap	仿真世界与真实世界在视觉和物理特性上的差异，是导致算法在真机上失效的主因。
领域随机化	Domain Randomization	在模拟中随机改变摩擦力、光照、颜色等参数，让机器人学会适应各种极端情况。
边缘案例	Corner Cases	极少发生但具有毁灭性的极端情况（如异常光照、微小碰撞），是系统鲁棒性的考验。
数字孪生	Digital Twin	现实物理实体在虚拟空间中的精准克隆，用于监控、预测和离线训练。

强化学习	Reinforcement Learning (RL)	通过 “试错” 获取奖励（Reward）来优化动作策略，类似训狗的智能进化方式。
奖励函数	Reward Function	定义任务目标的数学公式，告诉机器人哪些动作该鼓励，哪些动作该惩罚。
稀疏奖励	Sparse Rewards	任务极难时，机器人很难获得正向反馈，导致学习效率极低（如拧螺丝）。
在线适应	Online Adaptation	机器人在运行过程中，实时根据传感器反馈修正自身动作以适应环境变化的能力。
快速电机适应	RMA (Rapid Motor Adaptation)	一种让机器人（如机器狗）在 1 秒内通过感官历史感知地面特性并调整步态的技术。
神经物理仿真	Neural Physics	使用神经网络（而非硬写公式）来学习和预测复杂物理现象（如流体、软体）的技术。
世界模型	World Models	机器人对物理世界规律的内部理解，能让它在脑海中 “预演” 不同动作的后果。
鲁棒性	Robustness	系统的健壮性，指机器人在面对干扰、误差或未见过的情况时仍能正常工作的能力。

任务规划	Task Planning	将模糊的高层指令（如“我渴了”）拆解为机器人可执行的离散动作序列的过程。								
视觉-语言-动作模型	VLA Model (Vision-Language-Action)	一类端到端模型，直接输入图像和文字，输出机器人的控制指令（代表作：RT-2）。								
任务拆解	Task Decomposition	将一个长距离、复杂的任务细化为多个简单的“基元动作”（Primitives）。								
动作分词化	Action Tokenization	将连续的物理动作（如坐标、力矩）编码为离散的数字或字符（Tokens），让大模型像“写字”一样“写出”动作。								
常识推理	Commonsense Reasoning	机器人利用大模型在互联网学到的知识，理解“坏掉的电视”、“危险的刀”等抽象概念。								
零样本泛化	Zero-shot Generalization	机器人能够处理在训练阶段从未见过的物体或任务，依靠大模型的背景知识进行“跨界”应用。								
                                        							
SayCan	SayCan (LLM-based)	一种通过结合“语义评分 (Say)”和“物理可行性 (Can)”来进行任务选择的平衡算法。								
代码即策略	Code as Policies (CaP)	让大模型直接编写 Python 代码来调用机器人的 API，利用编程逻辑处理复杂任务。								
奖励工程	Reward Engineering	利用大模型（如 Eureka）自动编写复杂的奖励函数代码，代替人类专家进行强化学习指导。								
可执行度/功能性	Affordance	描述物体与智能体之间交互的可能性（例如：把手决定了杯子是“可抓取”的）。								
                                        							
样本效率	Sample Efficiency	衡量算法学习速度的指标。效率越高，意味着机器人通过越少的尝试就能学会复杂技能。								
探索与利用	Exploration vs. Exploitation	强化学习的核心博弈：是尝试全新的动作（寻找更高分），还是执行已知的稳妥动作（拿稳当前分）。								
奖励塑造	Reward Shaping	通过设计中间阶段的小奖励，引导机器人逐步走向最终目标，解决“方向性”问题。								
课程学习	Curriculum Learning	模仿人类教育，让机器人从极简单的任务开始练习，逐步增加难度（如：从走平地到走独木桥）。								
模仿学习	Imitation Learning	通过人类专家的演示数据进行初始训练，给强化学习提供一个良好的“起跑点”。								
RLHF	RLHF (RL from Human Feedback)	基于人类的主观反馈（如“动作很优雅”）来微调机器人的行为。
